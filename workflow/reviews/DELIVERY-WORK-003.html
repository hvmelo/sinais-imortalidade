M1 — readDir() swallows erros não-ENOENT

Arquivo: src/lib/content/loader.ts. O bloco catch {} retorna [] para qualquer erro de filesystem, não só "diretório não encontrado". Em produção, um problema de permissão causaria zero conteúdo sem erro visível.

// Fix (1 linha):
} catch (err) {
  if ((err as NodeJS.ErrnoException).code === 'ENOENT') return [];
  throw err;
}

Recomendação: incluir como Task 1 de Stage 1.4 — não requer hotfix nem review cycle.

NTH-1
isStringArray não rejeita strings vazias

src/lib/content/schema.ts — isStringArray aceita ["biotech", ""] sem erro, inconsistente com isString que exige .trim().length > 0. Latent bug se uma tag vazia entrar no frontmatter.

NTH-2
Nome de teste enganoso

loader.test.ts — 'loadSinais returns [] on empty directory' testa "retorna array", não "diretório vazio → vazio". Nome sugerido: 'loadSinais returns an array'.
